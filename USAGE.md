# 消息分段插件使用指南

## 快速开始

### 1. 安装插件

将 `astrbot_plugin_message_splitter` 文件夹复制到 AstrBot 的插件目录：

```
data/plugins/astrbot_plugin_message_splitter/
```

### 2. 重启 AstrBot

重启 AstrBot 或在 WebUI 中重载插件。

### 3. 验证安装

在聊天中发送一条消息，LLM 应该能够使用 `#split#` 关键字来分段发送消息。

## 配置选项

插件支持以下配置选项（在 `config.json` 中）：

```json
{
  "split_keyword": "#split#",
  "enabled": true
}
```

### 配置项说明

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `split_keyword` | string | `#split#` | 分段关键字 |
| `enabled` | boolean | `true` | 是否启用分段功能 |

## 使用场景

### 场景 1：多语言回复

**用户输入：**
```
请用中文和英文分别说你好
```

**LLM 回复：**
```
Hello!
#split#
你好！
```

**效果：**
- 第一个气泡：`Hello!`
- 第二个气泡：`你好！`

**效果：**

- 第二个气泡：`第二步：找到插件选项`

### 场景 3：代码和解释分离

**用户输入：**

## 高级配置

### 自定义分段关键字

如果你希望使用其他关键字，比如 `---` 或 `[[split]]`，只需修改配置文件：

```json
{
  "split_keyword": "---",
  "enabled": true
}
```

### 临时禁用功能

如果你想临时禁用分段功能，但不想卸载插件：

```json
{
  "split_keyword": "#split#",
  "enabled": false
}
```

## 注意事项

1. **关键字区分大小写**：`#split#` 和 `#SPLIT#` 是不同的
2. **仅处理 LLM 结果**：只有 LLM 生成的消息会被处理，普通指令回复不受影响
3. **自动清理空格**：插件会自动处理关键字前后的多余空格
4. **平台兼容性**：在某些平台上，分段后的消息可能仍显示为一条消息，这取决于平台适配器的实现

## 故障排除

### 问题：分段功能不工作

**解决方案：**
1. 检查 `enabled` 配置是否为 `true`
2. 确认 LLM 回复中确实包含了分段关键字
3. 查看 AstrBot 日志，确认插件已正确加载

### 问题：分段后的消息仍然连在一起

**解决方案：**
1. 这可能与你使用的消息平台有关
2. 某些平台（如某些 QQ 机器人实现）可能不支持真正的分段发送
3. 插件会在消息链中添加换行符作为分隔，但最终显示效果取决于平台

### 问题：LLM 不使用分段关键字

**解决方案：**
1. 确保在提示词中明确告诉 LLM 使用分段功能
2. 插件会自动在系统提示词中注入使用说明
3. 你可以手动在用户消息中提醒 LLM 使用 `#split#` 关键字

## 技术细节

### 工作流程

1. **LLM 请求阶段** (`on_llm_request`)
   - 插件拦截 LLM 请求
   - 在系统提示词中注入分段指令
   - 告知 LLM 如何使用分段关键字

2. **消息发送前** (`on_decorating_result`)
   - 检测 LLM 回复中的分段关键字
   - 将消息按关键字分割成多个部分
   - 重新构建消息链，在各部分之间添加分隔符

### 消息链结构

分段后的消息链结构如下：

```
[Plain("第一部分"), Plain("\n"), Plain("第二部分"), Plain("\n"), Plain("第三部分")]
```

每个部分都是独立的 `Plain` 组件，之间用换行符分隔。
